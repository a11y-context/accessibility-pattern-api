## Purpose
- Provide a practical methodology for validating that AI-generated code is accessible, not just “looks right.”

## What this will cover (stub)
- Minimum acceptance checks per component type
- Automated checks (example: axe-core / Playwright pipelines)
- Manual checks (keyboard, screen reader, focus order)
- Common failure modes in AI-generated UI
- How to turn failures into better prompts and better patterns
