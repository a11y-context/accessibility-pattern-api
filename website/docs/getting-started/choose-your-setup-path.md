## Pick your setup path
Choose the flow that matches how you want to enforce A11y Context guidance.

### Repo install (recommended)
- **Who owns it:** any developer or team
- **Time-to-value:** minutes
- **What it requires:** repo access; optional MCP access; optional docs indexing for fallback
- **Best for:** most teams; pilots; mixed tool environments

**Start here:** link to **Repo Install**.

### Admin-managed install (TBD)
- **Who owns it:** platform/admin/security
- **Time-to-value:** longer (policy + configuration)
- **What it requires:** centralized instruction support in your chosen AI client(s) + approval
- **Best for:** standardizing enforcement org-wide when supported

**Read next:** link to **Admin-managed enforcement** (Guides).

## Quick comparison
- **Portability:** repo install works everywhere; admin-managed depends on tool support
- **Control:** admin-managed is centralized; repo install is per-repo
- **Consistency across assistants:** repo install is easier when multiple assistants are used
